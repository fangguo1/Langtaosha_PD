# 论文数据库系统 - 0413 架构简化版表说明

设计目标：统一作品 + 多来源记录 + 原始元数据沉淀 + 来源文件追溯。
支持 Langtaosha、bioRxiv、arXiv、PubMed 并存。

核心设计原则：
- `papers` 只存统一作品的 canonical 视图字段。
- 来源记录进入 `paper_sources`，允许一篇作品对应多个来源。
- 原始来源 metadata 统一进入 `paper_source_metadata`。
- 来源特异字段进入 `paper_source_metadata.normalized_json.source_specific`。
- 当前不参与核心检索的附加信息，不再单独建表，统一进入 `paper_source_metadata.normalized_json.common_normalized`。

暂时取消独立建模的表：
- `paper_texts`
- `venues`
- `paper_publications`
- `paper_versions`
- `paper_citations`
- `fields`
- `paper_fields`

说明：
- 所有 `SERIAL` / `BIGSERIAL` 字段均为数据库自动生成的内部主键，无需手动插入。
- 下文内容以当前 `database/schema.sql` 为准。

================================================================================
1. papers（统一作品主表）
================================================================================
功能描述：
- 每行代表一篇去重后的学术作品。
- 只保存 canonical 统一视图字段。
- 来源侧原始字段与来源特异字段不直接落在本表中，而由 `paper_sources` 与 `paper_source_metadata` 承载。

字段说明：
| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| `paper_id` | `SERIAL` PK | 论文内部主键 |
| `work_id` | `VARCHAR(200)` | 全局唯一标识符（UUID v7 格式，前缀 W），用于 Vector DB 关联、跨系统数据交换、API 对外接口 |
| `canonical_title` | `TEXT` | 统一标题，可空 |
| `canonical_abstract` | `TEXT` | 统一摘要，可空 |
| `canonical_language` | `VARCHAR(32)` | 统一语言标识，可空 |
| `canonical_publisher` | `VARCHAR(200)` | 统一出版方，可空 |
| `submitted_at` | `TIMESTAMP` | 统一提交时间，可空 |
| `online_at` | `TIMESTAMP` | 统一上线时间，可空 |
| `published_at` | `TIMESTAMP` | 统一发表时间，可空 |
| `merge_status` | `VARCHAR(32)` | 合并状态，默认 `single_source`，取值约定：`single_source` / `merged` |
| `created_at` | `TIMESTAMP` | 创建时间，默认 `CURRENT_TIMESTAMP` |
| `updated_at` | `TIMESTAMP` | 更新时间，默认 `CURRENT_TIMESTAMP`，由触发器自动更新 |
| `canonical_source_id` | `BIGINT` FK | 指向 `paper_sources.paper_source_id`，标记 canonical 优先来源，可空 |

约束与索引：
- `PRIMARY KEY (paper_id)`
- `FOREIGN KEY (canonical_source_id) REFERENCES paper_sources(paper_source_id) ON DELETE SET NULL`
- `UNIQUE INDEX idx_papers_work_id (work_id) WHERE work_id IS NOT NULL`（全局唯一标识符）
- `INDEX idx_papers_canonical_source_id (canonical_source_id) WHERE canonical_source_id IS NOT NULL`

补充说明：
- `work_id` 使用 UUID v7 格式（前缀 W），保证全局唯一性，格式如 `W019b73d6-1634-77d3-9574-b6014f85b118`。
  - 用于 Vector DB 向量与元数据的关联
  - 用于跨系统数据交换和迁移
  - 用于 API 对外接口（避免暴露内部自增 ID）
  - UUID v7 时间有序，索引性能接近自增 ID
- `canonical_source_id` 通过 `ALTER TABLE` 在 `paper_sources` 创建后追加，以解决循环依赖。
- `merge_status='merged'` 表示当前 canonical 视图由多个来源合并而成。

2. paper_keywords（论文关键词表）
================================================================================
功能描述：
- 存储论文结构化关键词。
- 支持任务、方法、数据集、指标、模型等多种关键词类型。
- 记录关键词来源和置信度权重。

字段说明：
| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| `paper_id` | `INTEGER` FK | 论文ID，外键到 `papers.paper_id` |
| `keyword_type` | `VARCHAR(50)` | 关键词类型 |
| `keyword` | `VARCHAR(200)` | 关键词内容 |
| `weight` | `REAL` | 权重/置信度，默认 `1.0`，取值范围 `0.0-1.0` |
| `source` | `VARCHAR(50)` | 关键词来源，`NOT NULL DEFAULT 'paper_metadata'`；如 `paper_metadata`、`manual`、`ai_extract`、`keyword_match`、`llm_tag`、`scispacy-en_core_sci_lg-generated` |
| `created_at` | `TIMESTAMP` | 创建时间，默认 `CURRENT_TIMESTAMP` |

约束与索引：
- `PRIMARY KEY (paper_id, keyword_type, keyword, source)`
- `FOREIGN KEY (paper_id) REFERENCES papers(paper_id) ON DELETE CASCADE`
- `CHECK (weight >= 0.0 AND weight <= 1.0)`
- `INDEX idx_pk_paper_id (paper_id)`
- `INDEX idx_pk_keyword_type (keyword_type)`
- `INDEX idx_pk_keyword (keyword)`
- `INDEX idx_pk_source (source)`
- `INDEX idx_pk_source_keyword (source, keyword)`
- `INDEX idx_pk_lower_keyword (lower(keyword))`
- `UNIQUE INDEX idx_pk_keyword_case_insensitive_unique (paper_id, lower(keyword_type), lower(keyword), source)`
- `INDEX idx_pk_paper_id_keyword_type (paper_id, keyword_type)`

关键词身份约定：
- 同一 `paper_id + keyword_type + keyword + source` 是主键维度，允许同一关键词来自多个来源。
- 同一 `paper_id + lower(keyword_type) + lower(keyword) + source` 必须唯一，避免同一来源内出现 `Machine Learning` / `machine learning` 这类大小写重复。

关键词类型约定：
- `task`
- `method`
- `contribution`
- `dataset`
- `metric`
- `library`
- `domain`
- `concept`
- `model`

3. paper_author_affiliation（作者-论文-机构表）
================================================================================
功能描述：
- 以 JSONB 数组形式保存论文作者及机构信息。
- 与 `papers` 为 1:1 关系。

字段说明：
| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| `paper_id` | `INTEGER` PK, FK | 论文ID，主键，同时外键到 `papers.paper_id` |
| `authors` | `JSONB` | 作者数组，保存姓名、排序、机构等信息 |

约束与索引：
- `PRIMARY KEY (paper_id)`
- `FOREIGN KEY (paper_id) REFERENCES papers(paper_id) ON DELETE CASCADE`
- `INDEX idx_paa_paper_id (paper_id)`
- `GIN INDEX idx_paa_authors ON authors`

JSON 结构示意：
```json
[
  {
    "sequence": 1,
    "name": "Author Name",
    "affiliations": ["Institution A"]
  }
]
```

8. paper_sources（来源记录表）
================================================================================
功能描述：
- 每条记录表示一篇作品在某个来源系统中的一次可追踪记录。
- 用于承载来源侧标题、摘要、标识符、链接、时间字段等“来源级信息”。
- 是多来源并存架构的核心表之一。
- 旧迁移里挂在 `papers` 上的来源级标识（如 `pubmed_id`、`semantic_scholar_id`）已迁移到这里。

字段说明：
| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| `paper_source_id` | `BIGSERIAL` PK | 来源记录内部主键 |
| `paper_id` | `INTEGER` FK | 关联到统一作品 `papers.paper_id` |
| `source_name` | `VARCHAR(64)` | 来源名，如 `langtaosha`、`biorxiv`、`arxiv`、`pubmed` |
| `platform` | `VARCHAR(64)` | 平台名，可空 |
| `source_record_id` | `VARCHAR(200)` | 来源内部唯一记录ID，可空 |
| `source_url` | `VARCHAR(500)` | 来源页面URL，可空 |
| `abstract_url` | `VARCHAR(500)` | 摘要页URL，可空 |
| `pdf_url` | `VARCHAR(500)` | PDF URL，可空 |
| `title` | `TEXT` | 来源侧标题，可空 |
| `abstract` | `TEXT` | 来源侧摘要，可空 |
| `publisher` | `VARCHAR(200)` | 来源侧出版方，可空 |
| `language` | `VARCHAR(32)` | 来源侧语言，可空 |
| `doi` | `VARCHAR(200)` | DOI，可空 |
| `arxiv_id` | `VARCHAR(50)` | arXiv ID，可空 |
| `pubmed_id` | `VARCHAR(100)` | PubMed ID，可空 |
| `semantic_scholar_id` | `VARCHAR(100)` | Semantic Scholar ID，可空 |
| `submitted_at` | `TIMESTAMP` | 来源侧提交时间，可空 |
| `online_at` | `TIMESTAMP` | 来源侧上线时间，可空 |
| `published_at` | `TIMESTAMP` | 来源侧发表时间，可空 |
| `updated_at_source` | `TIMESTAMP` | 来源侧最近更新时间，可空 |
| `version` | `VARCHAR(50)` | 来源记录版本号，可空，用于同一来源内版本比较 |
| `is_preprint` | `BOOLEAN` | 是否预印本，默认 `FALSE` |
| `is_published` | `BOOLEAN` | 是否正式发表，默认 `FALSE` |
| `is_primary_source` | `BOOLEAN` | 是否为 canonical 优先来源，默认 `FALSE` |
| `sync_status` | `VARCHAR(32)` | 同步状态，默认 `active`，约定：`active` / `deprecated` / `error` |
| `last_synced_at` | `TIMESTAMP` | 最近同步时间，默认 `CURRENT_TIMESTAMP` |
| `created_at` | `TIMESTAMP` | 创建时间，默认 `CURRENT_TIMESTAMP` |
| `updated_at` | `TIMESTAMP` | 更新时间，默认 `CURRENT_TIMESTAMP`，由触发器自动更新 |

约束与索引：
- `PRIMARY KEY (paper_source_id)`
- `FOREIGN KEY (paper_id) REFERENCES papers(paper_id) ON DELETE CASCADE`
- `UNIQUE INDEX idx_ps_source_record_unique ON (source_name, source_record_id) WHERE source_record_id IS NOT NULL`
- `UNIQUE INDEX idx_ps_source_doi ON (source_name, doi) WHERE doi IS NOT NULL`
- `UNIQUE INDEX idx_ps_source_arxiv ON (source_name, arxiv_id) WHERE arxiv_id IS NOT NULL`
- `UNIQUE INDEX idx_ps_source_pubmed ON (source_name, pubmed_id) WHERE pubmed_id IS NOT NULL`
- `INDEX idx_ps_paper_id (paper_id)`
- `INDEX idx_ps_source_name (source_name)`
- `INDEX idx_ps_doi (doi)`
- `INDEX idx_ps_arxiv_id (arxiv_id)`
- `INDEX idx_ps_pubmed_id (pubmed_id)`
- `INDEX idx_ps_updated_at_source (updated_at_source)`
- `INDEX idx_ps_last_synced_at (last_synced_at)`

补充说明：
- 同一 `paper_id` 可以对应多条 `paper_sources`。
- `is_primary_source` 与 `papers.canonical_source_id` 共同服务于 canonical 来源选择逻辑。

9. paper_source_metadata（来源元数据沉淀表）
================================================================================
功能描述：
- 保存每条来源记录的完整原始 payload 和归一化结果。
- 是“原始数据可审计、归一化结果可重放”的核心表。
- 当前原本分散在 `paper_publications`、`paper_versions`、`paper_citations`、`fields` 等表的信息，统一进入 `normalized_json.common_normalized`。
- 旧迁移里的 `contribution_types`、`paper_type`、`primary_field`、`target_application_domain`、`is_llm_era`、`short_reasoning` 也统一沉淀到 `normalized_json.common_normalized`。

字段说明：
| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| `paper_source_id` | `BIGINT` PK, FK | 来源记录ID，1:1 对应 `paper_sources.paper_source_id` |
| `raw_metadata_json` | `JSONB` | 完整原始记录，非空 |
| `normalized_json` | `JSONB` | 归一化结果，结构约定为 `{"common_normalized": {...}, "source_specific": {...}}` |
| `parser_version` | `VARCHAR(64)` | 解析器版本，可空 |
| `source_schema_version` | `VARCHAR(64)` | 来源 payload 版本，可空 |
| `created_at` | `TIMESTAMP` | 创建时间，默认 `CURRENT_TIMESTAMP` |
| `updated_at` | `TIMESTAMP` | 更新时间，默认 `CURRENT_TIMESTAMP`，由触发器自动更新 |

约束与索引：
- `PRIMARY KEY (paper_source_id)`
- `FOREIGN KEY (paper_source_id) REFERENCES paper_sources(paper_source_id) ON DELETE CASCADE`
- `GIN INDEX idx_psm_raw ON raw_metadata_json`
- `GIN INDEX idx_psm_normalized ON normalized_json`

`normalized_json` 结构说明：
```json
{
  "common_normalized": {
    "title": "string",
    "abstract": "string",
    "language": "en",
    "publisher": "string",
    "authors": [],
    "categories": [],
    "keywords": [],
    "contribution_types": [],
    "paper_type": "survey",
    "primary_field": "biology",
    "target_application_domain": "genomics",
    "is_llm_era": false,
    "short_reasoning": "string",
    "pub_info": {
      "venue_name": "string",
      "venue_type": "journal",
      "publish_time": "2025-01-01",
      "presentation_type": null
    },
    "versions": [],
    "citations": {
      "cited_by_count": 0,
      "update_time": "2025-01-01"
    },
    "fields": []
  },
  "source_specific": {}
}
```

补充说明：
- `raw_metadata_json` 必须保留完整原始记录，不做裁剪。
- `common_normalized` 只放跨来源可解释的统一字段。
- `source_specific` 只放来源特异字段。

10. paper_references（来源参考文献表）

字段说明：

| `paper_source_id` | `BIGINT` FK | 来源记录ID，可空 |
| `reference_order` | `INTEGER` | 参考文献在原文中的顺序，可空 |
| `reference_text` | `TEXT` | 参考文献原文，非空 |
| `reference_raw_json` | `JSONB` | 来源提供的结构化参考文献数据，可空 |
| `created_at` | `TIMESTAMP` | 创建时间，默认 `CURRENT_TIMESTAMP` |

约束与索引：
- `PRIMARY KEY (reference_id)`
- `FOREIGN KEY (paper_id) REFERENCES papers(paper_id) ON DELETE CASCADE`
- `FOREIGN KEY (paper_source_id) REFERENCES paper_sources(paper_source_id) ON DELETE CASCADE`
- `INDEX idx_pref_paper_id (paper_id)`
- `INDEX idx_pref_paper_source_id (paper_source_id)`

11. paper_source_artifacts（来源文件追溯表）
================================================================================
功能描述：
- 记录来源数据文件或页面快照的存储位置与校验信息。
- 支持原始文件追溯、校验、重放和问题定位。

字段说明：
| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| `artifact_id` | `BIGSERIAL` PK | Artifact 内部主键 |
| `paper_source_id` | `BIGINT` FK | 来源记录ID |
| `artifact_type` | `VARCHAR(64)` | 文件类型，如 `raw_jsonl`、`manifest`、`raw_page`、`pdf` |
| `storage_backend` | `VARCHAR(32)` | 存储后端，如 `local`、`s3`、`oss` |
| `storage_uri` | `VARCHAR(1000)` | 可迁移存储 URI |
| `line_no` | `INTEGER` | 文件内行号，可空 |
| `byte_offset` | `BIGINT` | 文件内字节偏移，可空 |
| `sha256` | `VARCHAR(64)` | 文件内容校验值，可空 |
| `captured_at` | `TIMESTAMP` | 捕获时间，默认 `CURRENT_TIMESTAMP` |
| `created_at` | `TIMESTAMP` | 创建时间，默认 `CURRENT_TIMESTAMP` |

约束与索引：
- `PRIMARY KEY (artifact_id)`
- `FOREIGN KEY (paper_source_id) REFERENCES paper_sources(paper_source_id) ON DELETE CASCADE`
- `INDEX idx_psa_paper_source_id (paper_source_id)`
- `INDEX idx_psa_storage_backend (storage_backend)`
- `INDEX idx_psa_artifact_type (artifact_type)`
- `INDEX idx_psa_sha256 (sha256) WHERE sha256 IS NOT NULL`

补充说明：
- 推荐使用 `storage_backend + storage_uri + sha256` 组合做文件级追溯。
- 一条来源记录可对应多个 artifact。

附注：辅助函数
================================================================================
